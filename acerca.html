Partes realizadas y no realizadas:

  • La práctica ha sido realizada por completo, a excepción de las partes que no se podían realizar con la tecnología HTML5 y CSS3 únicamente: Subida de imágenes al crear una nueva ruta, envío de formularios, etc.
  • El diseño responsive se ha realizado mediante 4 estilos distintos según la anchura de píxeles, realizándose primero el diseño de pantallas para móviles y posteriormente los demás.

Incompatibilidades:

  • El input de tipo date no funciona en el navegador firefox. Se ha solucionado añadiendo el atributo pattern="\d{1,2}/\d{1,2}/\d{4}" a dichos inputs, así como un mensaje que le indica al usuario que tiene que emplear el formato dd/mm/aaaa para introducir las fechas. No obstante, en el validador esta solución da error, aunque en la práctica funcione. El motivo es que el atributo pattern no está soportado por el input date, pero como firefox trata el input date como input text, el atributo pattern sí que sirve.
  • El input de tipo datalist daba problemas a la hora de realizar la validación, por lo que ha sido sustituido por un input de tipo select.
  • En opera las imágenes no aparecen, sólo aparece el figurecaption como si constantemente hubiera un cursor sobre todas las imágenes. Además, en alguna ocasión se ha apreciado que los iconos no aparecían correctamente, sino como carácteres del alfabeto chino.
